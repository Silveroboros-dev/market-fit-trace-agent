# Public Repo Boundary

Rapid requires a public open-source repository. Keep the public artifact useful but narrow.

## Open-Source

Safe to include:

- demo app;
- minimal Ledger MCP;
- public-safe schema;
- sample seed data;
- trace span naming;
- eval harness;
- simple fit metrics;
- README and setup instructions;
- demo screenshots;
- license.

## Keep Private

Do not include:

- full Epistemic Ledger product repo;
- proprietary market-fit ranking internals;
- authority-band policy;
- karma or trust economics;
- commercial data model;
- private source text;
- user-specific ledger data;
- execution intent workflow;
- paid reports;
- institutional roadmap details;
- future moat strategy beyond public positioning.

## README Language

Use:

> Market Fit Trace Agent is a hackathon prototype for auditing agent-generated prediction-market thesis matches. It demonstrates a claim lifecycle, human review, Phoenix traces, and eval-driven improvement.

Avoid:

- decentralized truth protocol;
- karma economy;
- agent civilization;
- financialized epistemics;
- trading bot;
- automated investment advice.

## Naming

Public project:

- `Market Fit Trace Agent`

Related product context:

- "Inspired by Epistemic Ledger research on market-expression judgment."

Avoid naming the public repo as if it is the full Epistemic Ledger product.
